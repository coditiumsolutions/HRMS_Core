# HRMBT Constitution

## Purpose
Build a modular HR Management System that can grow over time.

## Tech Stack
- ASP.NET Core MVC
- SQL Server
- Entity Framework Core
- Bootstrap
- Cursor IDE

## Core Principles
- One module at a time
- Each module independently usable
- Specification is the source of truth
- No module breaks existing modules

## Non-Goals
- No premature optimization
- No unnecessary abstractions

## Definition of Done
A module is done when:
- It matches its specification
- It passes independent testing
- It does not break existing modules
